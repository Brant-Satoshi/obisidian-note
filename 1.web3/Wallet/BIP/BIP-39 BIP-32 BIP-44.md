## BIP 是什么

**BIP（Bitcoin Improvement Proposal，比特币改进提案）**是给比特币生态提出变更或标准化建议的公开文档流程。任何人都可以提交，用来：

- 规范**协议/网络/钱包/编码**等方面的改进；
    
- 给社区一个**可讨论、可审查、可实现**的共同依据。

比特币去中心化、参与者众多——BIP 提供一套**公开、版本化、可追踪**的协作机制，避免“各自实现、互不兼容”的混乱。

## 分类（常见三类）

- **Standards Track**：会影响实现或互操作性的技术标准（最核心，含共识、P2P、钱包格式等）。
    
- **Informational**：提供信息或最佳实践，不强制。
    
- **Process**：改进流程本身（治理、发布节奏、提交流程等）。

## 生命周期（简化）

`Draft（草案） → Proposed/Review（讨论/审查） → Final（定稿/被采用）`

- 也可能 **Rejected/Withdrawn/Obsolete/Superseded**（被否、撤回、过时、被替代）。
    
- 讨论主要在开发者邮件列表与 GitHub 上，由 **BIP 编辑**维护编号与状态。

## 和以太坊的 EIP 对比

- **BIP** 是比特币生态的标准流程；
    
- **EIP**（Ethereum Improvement Proposal）是以太坊对应的提案体系；思路相同，细节不同。

**BIP-39 = 助记词与种子**，**BIP-32 = 分层确定性(HD)密钥派生**，**BIP-44 = 多币种/多账户的标准派生路径格式**

## BIP-39（助记词）

- 定义：把随机熵编码成 **12/15/18/21/24 个英文单词** 的助记词，并用 PBKDF2 派生出 **seed（512bit）**。
    
- 口令（passphrase）：可选的“第25个词”，改变 seed（不同口令＝完全不同钱包）。
    
- 作用：**备份/跨钱包可迁移**；泄露助记词或口令就等于泄露全部资产。
